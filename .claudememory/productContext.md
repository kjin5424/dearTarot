# productContext.md (PixiJS 버전)

## 제품 정의

dearTarot은 타로 리딩 도구가 아니라 **픽셀아트 게임 형식의 자기 성찰 경험**이다.
유저는 타로를 "보는" 것이 아니라 어두운 숲 속 마녀 집회에 "참가"한다.

### 게임 형식

- **시점:** Top-down view (탑다운)
- **조작:** 유저가 화살표로 직접 이동하지 않음. 소녀의 안내를 따라가는 형식
- **향후:** 유저 직접 이동 가능성 있음 (현재는 비활성)

### 유저가 할 수 있는 행동

- DialogBox에 고민 내용 작성
- 소녀의 질문에 선택지로 대답
- 카르마 옵션 선택
- 카드 선택 (직관적 드로우)
- 카드 해석 내용 넘기기 (다음 버튼)
- 이메일 입력
- 공유 버튼 누르기

## UX 핵심 원칙

### 1. 게임이지 웹앱이 아니다

- URL이 바뀌지 않는다. 뒤로가기 버튼이 없다.
- 모든 전환은 씬(Scene) 전환이다. 페이지 이동이 아니다.
- 유저는 흐름에 실려간다. 내비게이션을 스스로 제어하지 않는다.

### 2. 픽셀아트 컨셉 일관성

- 모든 시각 요소는 픽셀아트 스타일을 유지한다
- Vue overlay (입력창, 모달, AI 패널)도 픽셀 폰트 / 픽셀 테두리 적용
- PixiJS: `PIXI.TextureSource.defaultOptions.scaleMode = 'nearest'` 전역 설정
- 해상도: 480×270 (16:9 저해상도, 화면에 맞게 정수 배율 스케일)
- PixiJS + GSAP 조합으로 Phaser보다 더 부드러운 트윈 애니메이션 가능

### 3. 소녀가 안내자다

- 모든 씬 전환의 "이유"는 소녀의 행동과 대사로 설명된다
- 유저가 다음 버튼을 누르는 것이 아니라 소녀의 안내를 따르는 것처럼 느껴야 한다
- 대화는 `DialogBox.vue`로 타이핑 효과 출력 (Vue 컴포넌트로 처리)
- 소녀 스프라이트(`WitchGirl.ts`)는 PixiJS AnimatedSprite로 걷기/말하기/손흔들기 표현

### 4. 카르마 교환의 진정성

- 카르마 정화는 귀찮은 절차가 아니라 의례(ritual)처럼 느껴져야 한다
- 강요하지 않되, 분위기로 자연스럽게 참여를 유도
- 세 가지 옵션 모두 동등하게 유효한 선택지

### 5. AI 해석은 단정하지 않는다

- "당신은 ~입니다"가 아니라 "이 그림은 ~를 보여줍니다"
- 카드의 그림에서 시작해서 유저 스스로 연결하도록 여백을 남김
- 스트리밍으로 천천히 출력하여 읽히는 느낌 강조

## UX 구현

| 요소               | Phaser 버전       | PixiJS 버전                   |
| ------------------ | ----------------- | ----------------------------- |
| 대화창(DialogBox)  | Phaser Text 객체  | Vue 컴포넌트 overlay          |
| 카드 선택 인터랙션 | Phaser 이벤트     | PixiJS eventMode + GSAP hover |
| 카메라 이동        | camera.pan()      | GSAP → Container.y            |
| 씬 전환 효과       | Phaser transition | GSAP alpha/scale 트윈         |
| 파티클 효과        | Phaser Particles  | PixiJS ParticleContainer      |
| 오디오             | Phaser Audio      | Howler.js (훅으로 래핑)       |

## 상세 내러티브 플로우

### FOREST_INTRO
유저는 어두운 밤, 침엽수가 빽빽한 숲의 좁은 일방통행 길 시작점에 서 있다.
"당신은 길을 잃었습니다. 나아갈 길을 찾고 싶으나 어두워 눈앞이 보이지 않습니다."
카메라가 y축 위로 서서히 이동하며 길을 따라간다.

### WITCH_CIRCLE
길을 따라가면 나무가 비어있는 공터가 나온다. 풀과 작은 꽃들만 자라있고, 신비로운 달빛을 받고 있다.
중간중간 구름과 나무 그림자로 가려진 곳도 있다.
공터 가운데에 **8명의 양머리 소녀**들이 손을 잡고 원을 그리며 빙글빙글 돌고 있다.

### WITCH_APPROACH
한 소녀가 유저를 발견하고 유저 쪽으로 걸어온다.
- "왜 이런 곳에 있어? 길을 잃었어?"
- "어쩌다 여기에 오게 되었는지 기억나?"
- "마음 속에 안개가 낀 사람들이 종종 이곳으로 길을 잃곤 해. 내가 돌아갈 길을 찾아줄게."
→ 유저에게 고민 입력 유도 (QUESTION_INPUT)

### QUESTION_INPUT → SPREAD_QUIZ
소녀 무리에게 걸어가면 소녀들이 대화를 시작.
- "바로 돌아가고 싶어? 우리랑 좀 더 있지 않을래?"
- 바로 돌아가고 싶다 → **원카드** 스프레드
- 좀 더 있겠다 → 추가 대화로 **쓰리카드 / 켈트십자가** 결정
대화를 통해 스프레드 방식과 셔플 방식을 자연스럽게 정한다.

### KARMA
"너가 돌아갈 길을 알려줄 카드를 골라볼게" → 카드 셔플 애니메이션 시작.
소녀가 옆에서 설명: "점술은 카르마를 쌓는 행위, 선의의 교환이 필요해."
복채 옵션 3가지 제시: 감사 일기 작성 / 후원 링크 / 광고 시청.

### SHUFFLE → DRAW
유저가 복채 옵션 선택 후 카드 선택 화면.
선택하면 스프레드 위치에 맞게 카드 배치 + 뒤집기 애니메이션.

### READING
순서대로 카드를 설명하고 타로 상담.
"이건 예언이 아니라 네가 앞으로 이 숲을 어떻게 빠져나가야 할지에 대한 조언일 뿐이야."

### RETURN
구름이 움직이며 그림자로 가려져 있던 반대편 길이 보인다.
소녀가 유저를 그 길로 안내해주고, 멀어지는 유저를 향해 손을 흔들며 끝.

## 윤리 필터 UX 원칙

- 거절하되 창피주지 않는다
- "그 질문은 안 돼요" 대신 "이런 방향으로 생각해보는 건 어떨까요?"
- 소녀의 말투로 부드럽게 전달 (EthicsModal.vue)
- 필터링 후에도 유저가 계속 진행할 수 있도록 대안 질문 제시

---

## 카드 데이터 원칙

- 전체 78장 (메이저 22장 + 마이너 56장)
- 정방향 70% / 역방향 30% 확률
- 역방향 카드는 PixiJS `sprite.angle = 180`으로 표현
- 해석은 카드 이미지의 시각적 요소 중심으로 서술

---

## 오디오 UX (Howler.js)

- 앰비언트 사운드는 씬 enter() 시 `fadeIn()` 처리
- 씬 exit() 시 `fadeOut()` 후 다음 사운드 `fadeIn()` (crossfade)
- 카드 뒤집기, 셔플 효과음은 8bit 계열 픽셀 게임 감성
- 소리 off 토글 옵션 제공

---

## 접근성 고려사항

- 텍스트 대화는 클릭으로 스킵 가능
- 인트로 씬은 스킵 버튼 제공 (재방문 유저 배려)
- AI 해석 텍스트는 복사 가능하도록 처리 (Vue overlay이므로 기본 지원)
- PixiJS canvas는 aria-label 설정, 주요 조작은 Vue DOM으로 처리해 접근성 확보

---

## 향후 확장 고려 (현재 구현 범위 아님)

- 리딩 히스토리 뷰 (localStorage 기반)
- 소녀 캐릭터 커스터마이징
- 멀티 언어 지원 (한/영/일)
- PixiJS 필터 효과로 야간 분위기 강화 (CRTFilter, GlitchFilter 등)
- 향후 유튜브 등 타 플랫폼 채널과의 연계를 고려한 마케팅 파이프라인
- 웨이트 타로를 넘어 오라클, 점성술 등으로 확장 가능한 **Generic Card JSON Schema** 설계
- Flutter 모바일 앱과의 데이터 호환성 보장
- 상담 결과를 잊지 않도록 정기적인 리마인드 이메일 발송.
- 이전 기록과 현재를 비교하여 유저의 성장을 시각화하는 대시보드 제공.
- SNS 공유 시 가장 예쁘게 보일 수 있는 '오늘의 카드 결과지' 이미지 생성 기능
